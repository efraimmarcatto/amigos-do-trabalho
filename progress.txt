## Codebase Patterns
- No `godot` CLI available in this environment ‚Äî typecheck cannot be run automatically
- Git config has empty `gpg.format` ‚Äî use `GIT_CONFIG_GLOBAL=/dev/null` with `-c user.name` and `-c user.email` flags for commits
- GlobalInput plugin at `addons/global_input/plugin.gd` is a `@tool` EditorPlugin ‚Äî autoload registration was removed; _enter_tree/_exit_tree now just `pass`
- In the shop buy popup, `QtyLabel` and `MinusBtn` are inside a nested HBoxContainer ‚Äî use `find_child()` not `get_node()` for nested nodes
- For shop/inventory grid icons: use `PRESET_FULL_RECT` + `EXPAND_IGNORE_SIZE` + `STRETCH_KEEP_ASPECT_CENTERED` + `clip_contents = true` on parent Button ‚Äî avoids misalignment and cropping
- Bubble is child of pet node ‚Äî inherits `scale.x` flip. Counter-scale with `_bubble_panel.scale.x = signf(scale.x)` in `_process()` and mirror `position.x` accordingly
- `get_bubble_rect()` must use `to_global()` not `global_position + local_position` to correctly handle parent transform/scale
- Root Control nodes (child of Node2D) may have `size == (0,0)` ‚Äî use `get_combined_minimum_size()` as fallback for passthrough rect calculations
- When emitting visibility signals that trigger passthrough updates, use `call_deferred()` so the layout pass computes panel size first
- `_furniture_nodes` and `_furniture_positions` use instance keys (e.g., `sofa_1`) ‚Äî use `_get_furniture_type()` to extract the base furniture ID, and `_generate_instance_key()` to create new keys
- Inventory enforces placement limits: `remove_from_inventory()` is called before placement and returns false if count is 0
- Save/load migrates old bare furniture keys to instance keys on load ‚Äî use `_is_instance_key()` to detect format

## 2026-03-24 - US-006
- Fixed mood bubble being invisible on Windows unless overlapped by furniture
- Root cause: `_bubble_panel.size` returns `(0, 0)` for root Control nodes (PanelContainer as child of Node2D) because the layout system doesn't auto-compute size for Controls outside the Control tree
- Added fallback to `get_combined_minimum_size()` in `get_bubble_rect()` when `size` is zero ‚Äî this correctly computes size from children's minimum sizes
- Deferred `_update_passthrough()` call in `_on_mood_bubble_changed()` so the panel layout pass runs before the passthrough polygon is rebuilt
- Files changed: `scripts/pet.gd`, `scripts/main.gd`
- **Learnings for future iterations:**
  - Control nodes parented to Node2D (root Controls) may not have their `size` property auto-computed ‚Äî always check for zero size and fall back to `get_combined_minimum_size()`
  - On Windows, `mouse_passthrough_polygon` determines visual opacity too ‚Äî any area not covered by the polygon is fully transparent to the compositor
  - When a signal triggers a layout-dependent calculation, defer the call so the layout pass runs first
