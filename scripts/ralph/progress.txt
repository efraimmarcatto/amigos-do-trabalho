## Codebase Patterns
- Use `GIT_CONFIG_GLOBAL=/dev/null` with `-c user.name=... -c user.email=...` to work around broken gpg.format in .gitconfig
- No typecheck tooling available for this Godot project — verify GDScript syntax manually
- FurnitureData is a Resource class in scripts/furniture_data.gd; runtime node is Furniture in scripts/furniture.gd
- Furniture Area2D `input_pickable` must be false — it's only for collision detection
- Existing furniture .tres files: bookshelf, food_bowl, sofa, table, toy in furniture/data/
- collision_size_override and collision_offset default to Vector2.ZERO — when zero, auto-sizing behavior is used
- Branch created from ralph/layout-edit-shop-pet-regression-fixes (has all furniture infrastructure)
- Plugin panel layout: HSplitContainer with LeftPanel (form, 60%) and RightPanel (preview, 40%) in furniture_creator_panel.tscn
- All editor plugin scripts must use @tool annotation
- Form fields built programmatically in _build_form(); use _add_field_row() helper for label+field rows
- interaction_type OptionButton indices: 0=(none), 1=play, 2=eat, 3=sleep
- Use EditorFileDialog (not FileDialog) in editor plugins; use set_meta/get_meta to pass state to custom draw controls
- Sprite source uses TabContainer with tabs "From Atlas" (index 0) and "From File" (index 1); use get_selected_texture() for unified access
- Preview uses PreviewDisplay (preview_display.gd) custom Control with _draw() for texture + collision overlay compositing
- Collision getters: get_collision_size() returns Vector2.ZERO for auto, get_collision_offset() returns Vector2.ZERO when unchecked
- Standing getters: get_standing_size() returns Vector2.ZERO for auto, get_standing_offset() returns Vector2.ZERO when unchecked — same pattern as collision
- Conditionally-visible form rows: store row reference from _add_field_row(), or use control.get_parent() for rows built as local vars
- Save .tres files: ResourceSaver.save(resource, path); refresh with EditorInterface.get_resource_filesystem().scan()
- Check file existence with FileAccess.file_exists() for res:// paths; use ConfirmationDialog for overwrite prompts
- Load .tres resources: `load(path)` returns Resource; check `is FurnitureData` before casting
- Detect atlas vs standalone textures: `data.texture is AtlasTexture`; get source sheet via `.atlas.resource_path`
- Preview drag interactions: use _gui_input method override, DragTarget enum for state, emit signals on mouse release to sync form spinboxes
