## Codebase Patterns
- coin_hud.gd `setup()` signature: `(floor_y, toggle_x, toggle_w)` — no menu_panel_h param
- slide_menu.gd declares `menu_opened` and `menu_closed` signals but they are not currently connected in main.gd
- When removing unused code, grep all `.gd` files to confirm no remaining references before committing
- Furniture display_scale is accessed as `fnode.data.display_scale` (a Vector2 on FurnitureData)
- All furniture is spawned as children of `_furniture_container` (Node2D, z_index=0), pet sprite has z_index=1
- During edit mode, ALL left mouse clicks/releases must be consumed in `_handle_edit_input` to prevent slide_menu's `_unhandled_input` from auto-closing the menu
- `_edit_mode_placement` flag in main.gd tracks placement initiated from edit mode — re-enters edit mode after placement
- Shop and PetSelection panels use GridContainer for icon grids; columns set dynamically based on panel width
- Shop buy popup uses named child nodes (ItemName, QtyLabel, CostLabel, BuyBtn, MinusBtn) accessed via `get_node()`
- Inventory action popup uses named child nodes (ItemName, PlaceBtn, DiscardBtn, CancelBtn) accessed via `get_node()`
- Inventory and Shop panels share the same grid cell pattern: VBoxContainer with Button (icon + overlay) inside
- ConfirmationDialog positioning: use `.position = Vector2i(x, y)` + `.popup()` instead of `.popup_centered()` for custom placement
- Stackable items use `_get_stacked_furniture_ids(base_id)` to find items on top — checks X bounds and Y proximity (8px tolerance)
- `_get_walkable_furniture_at_x(x)` returns the walkable Furniture node whose X bounds contain a given x coordinate
