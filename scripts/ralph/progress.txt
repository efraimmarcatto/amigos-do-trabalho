## Codebase Patterns
- UI panels (shop, coin HUD, etc.) are positioned programmatically in `setup()` using `floor_y` and `DisplayServer.screen_get_size()`
- All interactive UI must be included in the passthrough polygon via `_update_passthrough()` in main.gd — use `get_rect()` or `get_global_rect()` helpers
- Tween animations use 0.3s duration, `EASE_OUT`, `TRANS_CUBIC` — match this for consistency
- slide_menu.gd emits `menu_opened`/`menu_closed` signals for coordinating UI animations
- Adjacent panels (shop, inventory, settings) use `setup(menu_open_x, menu_panel_y)` for positioning relative to the slide menu
- Use `on_before_close: Callable` on slide_menu to intercept close and sequence panel close animations
- Panel open/close uses tween on `position:x` with `_animate_x()` pattern — hidden off-screen at `screen_w`, visible at computed `_open_x`
- When adding new adjacent panels, update these locations in main.gd: `_handle_menu_close()`, `_update_passthrough()`, `_on_edit_layout_requested()`, `_enter_placement_mode()`
- coin_hud.gd builds its UI programmatically in `_build_ui()` (no child nodes in .tscn)
- The CoinSystem emits `coins_changed(new_total: int)` — connect to this for coin display updates
- InventorySystem follows the same pattern as CoinSystem: extends Node, emits `inventory_changed(furniture_id, new_count)` signal
- JSON parses numbers as float — always handle both `is float` and `is int` when loading numeric values from save_data.json
- Inventory stored as `Dictionary[String, int]` mapping furniture_id to quantity; items with 0 count are erased from the dictionary
- Dynamic Button nodes in Node2D: position is relative to parent; `_input` must NOT call `set_input_as_handled()` for clicks on dynamically added Buttons
- To remove placed furniture: erase from `_furniture_nodes`, `_furniture_positions`, `queue_free()` the node, and check `pet_sprite._current_surface` for FALLING
- Translations loaded at runtime from CSV via `_load_translations()` in settings_panel.gd — no editor import step needed
- When adding new panels, also add mutual exclusivity closes in `_on_shop_button_pressed()`, `_on_inventory_button_pressed()`, and any new panel button handler
- Pet animations use convention names: idle, walk, jump_prep, jump, fall, interact, dragged — always check `has_animation()` before playing for graceful fallback
- Use `tr("KEY")` for translatable UI strings; `_update_all_ui_text()` in settings_panel.gd refreshes all panels on language change
- After monitor change, call `await get_tree().process_frame` before reading new screen dimensions, then reposition everything via setup() calls
- Furniture sprite is set in furniture.gd's _ready() method — apply display_scale there to both Sprite2D and CollisionShape2D
- All texture.get_size() calls for furniture must be multiplied by display_scale — check main.gd, pet.gd, and furniture.gd
- Pet SpriteFrames are pre-built .tres files — `has_animation()` before playing for graceful fallback
- `DisplayServer.get_screen_count()` is the correct API (not `screen_get_count()`)
