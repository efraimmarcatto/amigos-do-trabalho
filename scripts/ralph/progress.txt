## Codebase Patterns
- GlobalInput is a C++ GDExtension singleton registered via Engine::register_singleton in register_types.cpp — no autoload needed
- The .gitconfig has gpg.format that must be set to "openpgp" (was empty/broken)
- Project uses Godot 4.6 with GL Compatibility renderer and transparent viewport
- Save data stored at user://save_data.json — extend this for future features
- Pet visual states (SAD/NEUTRAL/HAPPY) use modulate color tinting on the Sprite2D — now called PetMood (separate from PetState movement enum)
- pet.gd uses PetState enum (IDLE, WALKING, FALLING, DRAGGED, INTERACTING) for movement; _process() delegates to _process_<state>() handlers
- main.gd is a Control node that manages passthrough polygon for click-through transparency
- _update_passthrough() collects Rect2 from all visible elements and merges them — add new UI elements there
- _change_state() in pet.gd handles enter/exit visual effects for state transitions — add new state visuals there
- Floor Y for pet landing = DisplayServer.screen_get_size().y - (texture height * scale.y) / 2.0
- Window is set to fullscreen at runtime via DisplayServer.screen_get_size() in main.gd _ready()
- When sprite flips (scale.x negative), use scale.abs() for size calculations in _is_point_on_pet() and _toggle_menu()
- _idle_timer in pet.gd controls wait between walks (1-5s random); _walk_target_x is the horizontal destination
- FurnitureData is a Resource with class_name — globally available, .tres files live in furniture/data/
- .tres resource files use [gd_resource script_class="FurnitureData"] format; load_steps = ext_resources + 1
- Shop (scripts/shop.gd) exposes get_owned()/set_owned() for persistence and furniture_purchased signal for spawning
- pet.gd _furniture_nodes is a shared Dictionary reference set by main.gd — new furniture spawns are automatically visible to pet
- _current_surface tracks which Furniture the pet is standing on (null = screen floor) — used in IDLE, WALKING, FALLING
- Furniture surface detection in FALLING uses prev_y/position.y crossing check to avoid tunneling through surfaces
